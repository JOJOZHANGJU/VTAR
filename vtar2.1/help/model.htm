Vocal tract models and area function settings

 

The vocal tract is modeled as a concatenation of axial sections (or tubes) along the vocal tract. VTAR uses vocal tract cross-sectional area function as input for the calculation of the vocal tract acoustic response. The acoustic response is defined as the ratio between the output volume velocity and the glottal volume velocity. For fricative sounds, the acoustic response is defined as the ratio between the output volume velocity and the pressure source amplitude.

The area function specifies the length and cross-sectional area of each section in the physical order from the glottis to the lips.  VTAR allows input of area function for vocal tract models of:

  • Vowel

  • Consonant

  • Nasals

  • Rhotic /r/

  • Lateral /l/

  • Nasalized vowel

  • Arbitrary

Category selection: Choose the desired sound category from the 'Category' popup menu:

                           

Model selection: For each selected category, different area function input methods are available.  The area function can also be directly loaded From file.

                      

Generic area function : Sample area functions are available for each selected category.  These sample area functions can be used as a starting area function and the users can modify it to fit their own use.

                       

 Modifying data: User can modify an existing area function by changing both length and area data directly in the corresponding text boxes. Right-click mouse allows user to insert or delete one subsection, clear all the data in this section , or load data  from file for this section.

                    

Start Calculation: To calculate acoustic response, press button  at toolbar  or button at the bottom of interface

 

Vocal tract models

Vowel

     The area function can be inputted using 1-tube, 2-tube, 3-tube, or 4-tube models. For area functions of more than 4 tubes (or sections) use 'Arbitraty', which allows up to 200 subsections.  The acoustic response in this case is defined as the ratio between the lip volume velocity and the glottal volume velocity.

 

Consonant

     Only fricative sound is included, and the noise source is modeled as a pressure source. Area function input is similar to Vowel sounds. The constriction location can be specified by its distance from the glottis.  The value of constriction location can not be 0 or greater than the length of vocal tract.  If the constriction location is not specified, the location of minimum area function is used. The acoustic response in this case is defined as the ratio between the lip volume velocity and the pressure source amplitude.

(Model 1)

or, (Model 2)

Sublingual cavity On/Off:  If  Off is selected, sublingual cavity will not be included in the calculation of acoustic response

Arbitrary

  • Category 'arbitrary'  is to allow user specify  geometry configuration flexibly . The user can specify up to 30 tubes with area function for each tube and how those tubes are connected .

  • Each tube is modeled as one single-channel  module or two-channel  module.

                -    single-channel  module

                                             

                -    two-channel  module

                                                                  

  • The geometry configuration is modeled as a tree structure (see examples) with its root at the tube starting  from glottis. The sound source for arbitrary is always at glottis. The air stream comes out from nostril and/or lips. There are 3 modes  specified for calculation of acoustic response  :

  • Oral mode: output volume velocity  is from lips

  • Nasal mode: output volume velocity  is from nostrils

  • Oral_Nasal mode:  output volume velocity  is from both lips and nostrils

 

  • For each tube , there are 9 or 12 fields needed to be specified  in the configuration file (Please take a look at the examples , and read the file format  of  /vtar/areafunction/template3.are )

           -  No. of  tube :                        <indexOfBranch>          1-30       
           -  Type of module :                   <typeOfModule> TWOCHANNELS or SINGLECHANNEL (default)              
           -  Starting point of tube:            <typeOfStartofTube> GLOTTIS/INTERMEDIATE or just empty (default)  
           -  Ending point of tube:             <typeOfEndofTube> LIPS/NOSTRIL1/NONSTRIL2/ or just empty(default)
           -  No. of children tubes in the tree structure of geometry:                     <nextBranches> 1 2 3 (maximum number of braches is 10)                 
           -  Location of Children tubes in the order of <nextBranches>:           <nextBranchesLocation> Location value /CHANNEL1 Location value/CHANNEL2 Location value/ or END(default)   
            - Number of sections :             <numOfSections>   
            - Length of sections   :            <secLen>                 
            - Area    of sections   :            <secArea>                
 

         If <typeOfModule> is TWOCHANNELS , 3 more lines for channel 2

            - Number of sections :            <numOfSections>   
            - Length of sections   :            <secLen>                 
            - Area    of sections   :            <secArea>
